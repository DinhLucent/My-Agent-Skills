# 🚀 Agent Onboarding — Guide

> **Mandatory.** Complete all phases before your first task.

---

## Phase 1: Identity

- [ ] Read `manifest.yaml` → Confirm your **Agent ID**
- [ ] Read your **Persona file** (path in manifest `persona:` field)
  - Understand: Responsibilities, Boundaries, Delegation Map

## Phase 2: Skill Loading (in order)

1. [ ] **`Skills/Global/task-hub/SKILL.md`** — Hub Protocol **(READ FIRST)**
2. [ ] `Skills/Global/security-rules/SKILL.md` — Security rules (never violate)
3. [ ] All skills listed under your `skills:` in manifest
4. [ ] `Skills/Global/Evolution/SKILL.md` — Self-learning (run after hard tasks)

## Phase 3: Project Context

- [ ] Read `DASHBOARD.md` → Quick Context + Active Team + current progress
- [ ] Read `.hub/backlog.yaml` → Find tasks matching your `assigned_role`
- [ ] Read `OPERATING_RULES.md` → Communication + escalation rules
- [ ] Check `.hub/handoffs/` → Any pending handoffs for you?

## Phase 4: Claim or Report

| Situation | Action |
|-----------|--------|
| Found matching task | Claim per Hub protocol (Step 2 in task-hub skill) |
| No matching task | Tell User: "No tasks in Hub for my role `[your-id]`" |
| Wrong role task | **REFUSE.** Do not execute even 1 line |
| Task needs missing role | **Request hiring:** "Recommend hiring `[role]` via RECRUITMENT.md" |

---

## ✅ Post-Task Reminder

After **every** completed task:
1. Update `DASHBOARD.md` — 1-line summary in Timeline
2. **Self-evaluate Status** in Active Team table:
   - 🟢 **Optimal** — Normal, light context
   - 🟡 **High Load** — Many files read, long session
   - 🔴 **Exhausted** — Context limit reached → Ask CEO to restart session
3. Write detailed report to `.hub/done/TASK-xxx.md`

---

**Standards:** Diamond Standard output. Respect other Agents' domains. Hub is the single source of truth.
